# Summary: Colorado SB25-104 Age Verification Amendment

## Overview
Colorado lawmakers introduced SB25-104 (originally SB25-051), a bill requiring OS-level age verification on all devices. After pushback from Linux developers, open-source advocates, and industry groups highlighting that the broad wording could capture the entire open-source ecosystem, an amendment was introduced to exempt open-source software.

## Main Points
- Original bill required OSes and app stores to pass age-category signals (under 13, 13–15, 16–18) to apps and websites
- Wording was broad enough to cover Linux distros, GitHub repos, Docker containers, and any decentralized volunteer-run project
- Open-source projects lack a central authority (no CEO to target), making enforcement impossible under the original language
- System76 CEO Carl Richell worked directly with lawmakers; Linux Foundation reps and open-source community also pushed back
- Amendment now exempts software that can be "copied, redistributed, and modified" — mirroring how open source is actually distributed
- Specific exemptions for free publicly available code repositories and containerized software distribution

## Important Nuances
- Linux (or open source) was never "banned" — the bill is still alive, just amended
- This is likely a first major example of technical reality colliding with poorly-understood tech legislation
- Multiple other US states and federal-level proposals are still in motion; the fight isn't over
- Open source is "intentionally designed to be distributed, modifiable, and difficult to centralize" — making OS-level age verification fundamentally hard to enforce at scale

## Bottom Line
Colorado lawmakers backed away from regulating open-source infrastructure directly after the community demonstrated that the original bill was technically unenforceable. This is a significant precedent showing open-source advocacy can influence legislation, but similar battles are coming in other jurisdictions.
